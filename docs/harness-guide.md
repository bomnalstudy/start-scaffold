# Harness Guide

하네스는 AI가 만든 결과물이 진짜로 안전한지 빠르게 검사하는 최소 검증 장치입니다.

## 하네스가 필요한 이유

AI는 그럴듯한 코드를 빠르게 만들지만, 아래 문제를 자주 남깁니다.

- 핵심 흐름은 안 돌았는데 된 것처럼 설명함
- 일부 케이스만 맞고 경계 조건이 깨짐
- 설정, 입출력, 상태 전환에서 회귀를 만듦

하네스는 이런 문제를 초기에 잡기 위한 "작고 반복 가능한 검증 프레임"입니다.

## 하네스의 목표

- 빠르게 실행 가능해야 함
- 실패 지점이 명확해야 함
- 핵심 시나리오를 우선 보호해야 함
- 사람이 읽기 쉬운 출력이 나와야 함

## 권장 구성

### 1. Scenario List

반드시 지켜야 할 핵심 흐름 목록입니다.

예시:

- 회원가입 성공
- 로그인 실패
- 토큰 갱신
- 저장 실패 시 롤백

### 2. Preconditions

시나리오 실행 전 필요한 상태입니다.

예시:

- 테스트 계정 존재
- 로컬 env 적재
- DB 시드 완료

### 3. Actions

시나리오에서 실제로 수행할 단계입니다.

예시:

- API 요청 전송
- 버튼 클릭
- 파일 생성

### 4. Assertions

무엇이 참이어야 통과인지 정의합니다.

예시:

- 응답 코드 200
- 에러 메시지 일치
- 파일 생성 안 됨
- 상태값 변경 없음

### 5. Output

결과는 짧고 명확해야 합니다.

예시:

- pass / fail
- 실패한 단계
- 재현에 필요한 최소 정보

## 하네스 레벨

### Level 1. Prompt Harness

문서 기반 체크리스트만으로 검증합니다.

언제 쓰나:

- 프로젝트 초기
- 코드가 아직 적을 때
- 빠른 설계 검토

### Level 2. Script Harness

PowerShell, Node, Python 등으로 반복 검증 스크립트를 만듭니다.

언제 쓰나:

- 같은 검증을 여러 번 반복할 때
- 팀원이 여러 환경에서 확인해야 할 때

### Level 3. Test Harness

테스트 러너나 E2E 프레임워크와 연결합니다.

언제 쓰나:

- 핵심 흐름이 이미 안정화 단계일 때
- 회귀 방지가 중요한 프로젝트일 때

## 바이브 코딩에 맞는 최소 하네스 전략

처음에는 아래 세 가지만 있어도 충분합니다.

1. 핵심 시나리오 3개
2. 실패하면 안 되는 조건 3개
3. 수동 또는 스크립트 검증 절차 1개

## 예시 하네스 설계

대상:

- 로그인 기능

시나리오:

- 올바른 계정 로그인 성공
- 잘못된 비밀번호 로그인 실패
- 만료된 토큰 재발급 실패 처리

검증:

- 성공 시 홈으로 이동
- 실패 시 정의된 메시지 출력
- 만료 토큰은 세션 무효화

## 오케스트레이션과 연결하는 법

- Planner가 하네스 범위를 먼저 정의합니다.
- Builder는 하네스를 깨지 않는 범위에서 수정합니다.
- Verifier는 하네스를 실행합니다.
- Recorder는 실패 케이스와 누락된 시나리오를 기록합니다.

## 하네스 작성 원칙

- 전체 기능이 아니라 핵심 위험을 우선 보호합니다.
- 실패 메시지는 사람이 바로 이해할 수 있게 씁니다.
- 재현 경로를 남깁니다.
- 새로운 버그를 잡았으면 시나리오를 추가합니다.

Research-backed 참고:

- Playwright isolation and user-visible behavior:
  https://playwright.dev/docs/best-practices
- pytest fixture modular setup:
  https://docs.pytest.org/en/stable/how-to/fixtures.html

추가 적용:

- 시나리오 간 상태 공유를 기본 금지합니다.
- 외부 의존성은 가능한 한 제어된 응답으로 고정합니다.
- 사전조건(데이터/세션/설정)을 명시적으로 선언합니다.

## 권장 질문

하네스를 만들기 전 아래를 먼저 정리합니다.

- 이 기능이 망가지면 가장 큰 피해는 무엇인가
- 반드시 지켜야 하는 사용자 흐름은 무엇인가
- 가장 자주 깨질 수 있는 경계 조건은 무엇인가
- 가장 싼 비용으로 어떤 검증을 자동화할 수 있는가
