# Workflow

## 추천 일일 흐름

1. 작업 목표를 한 줄로 정의
2. 관련 문서와 파일만 열기
3. AI에 작은 범위로 요청
4. 수정 후 즉시 검증
5. 결과를 `worklogs/`에 기록
6. 다음 세션을 위한 TODO 정리

## 새 기능 시작 전

- 범위가 한 문장으로 닫히는가
- 영향 범위를 알고 있는가
- 완료 기준이 있는가
- 검증 방법이 있는가

## 버그 수정 전

- 재현 조건이 있는가
- 원인 후보를 1~3개로 좁혔는가
- 로그/에러 메시지를 확보했는가
- 수정 후 재현이 사라지는지 확인 가능한가

## 세션 종료 전

- 문서 갱신 여부 확인
- 개발 일지 작성
- 불필요한 임시 파일 정리
- 더 이상 쓰지 않는 파일은 무력화 후 `.graveyard/`로 이동
- 다음 작업 1~3개 남기기
- 빈 worklog/task 템플릿, `tmp`/`temp`/`scratch`/`debug` 성격의 파일이 커밋 대상에 남지 않았는지 확인
