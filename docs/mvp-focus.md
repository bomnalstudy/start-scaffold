# MVP Focus (Speed First)

This document defines the current project mode.

## Primary Goal

- Save time by increasing delivery speed.

## Secondary Goal

- Improve output quality only when it reduces rework time.

## Success Metric

- We finish small, useful increments faster.
- We stop on time and avoid endless sessions.
- We reduce repeated explanation and repeated fixes.

## Current MVP Scope

1. Start gate:
   - Do not start coding without `Original Goal`, `MVP Scope`, `Done When`, `Stop If`.
2. Close gate:
   - Do not end session without stop rationale and 1-3 next tasks.
3. Minimal context loading:
   - Load only required context pack/files.

## Deferred (Phase 2+)

- Full strict checkpoint on every turn.
- Advanced orchestrator/harness automation.
- Extra token analytics dashboards.
- Plugin-first UX expansion.

## Operating Rule

When there is a tradeoff, choose the option that saves more time today
without creating obvious rework tomorrow.
